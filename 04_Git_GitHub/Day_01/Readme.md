# 📖 Introduction

Git is a **Distributed Version Control System (DVCS)** used to track changes in source code, collaborate with developers, and maintain different versions of a project.

Git helps developers:

- Track code changes
- Restore previous versions
- Collaborate with teams
- Maintain project history
- Manage different features using branches

GitHub is a cloud platform used to host Git repositories online.

# 📚 Topics Covered

Today I learned how to inspect changes and view commit history using Git.

# 1️⃣ git diff

## Definition

The `git diff` command shows the differences between files in the **Working Directory** and the **Staging Area**.

It helps developers review changes before committing them.

---

## Syntax

```bash
git diff
```

---

## Example

Before

```text
Hello this is Before Git Command (git diff) used
```

After editing

```text
Hello this is After Git Command (git diff) used
```

Running

```bash
git diff
```

Output

```diff
-Hello this is Before Git Command (git diff) used
+Hello this is After Git Command (git diff) used
```

---

## Symbols

| Symbol | Meaning |
|---------|----------|
| `+` | Added Line |
| `-` | Removed Line |
| `@@` | Position of the changes |

---

# 2️⃣ git diff --staged

## Definition

Displays the differences between the **Staging Area** and the **Last Commit**.

Useful for reviewing exactly what will be committed.

---

## Syntax

```bash
git diff --staged
```

or

```bash
git diff --cached
```

---

# 3️⃣ git log

## Definition

Displays the complete commit history of the repository.

Every commit contains:

- Commit Hash
- Author
- Date
- Commit Message

---

## Syntax

```bash
git log
```

---

## Sample Output

```text
commit 142066b4bf109f9c58bf9b093c0a6630380617c4
Author: Mohamad Zaid
Date: Sat Jul 4 2026

Learning Git Diff
```

---

# Commit Components

## Commit Hash

A unique identifier generated by Git for every commit.

Example

```text
142066b4bf109f9c58bf9b093c0a6630380617c4
```

---

## Author

Shows who created the commit.

---

## Date

Shows when the commit was created.

---

## Commit Message

A short description explaining the purpose of the commit.

Example

```text
Learning Git Diff
```

---

# 4️⃣ git log --oneline

## Definition

Displays a compact version of the commit history.

Instead of showing all commit details, it displays:

- Short Commit Hash
- Commit Message

---

## Syntax

```bash
git log --oneline
```

---

## Example

```text
142066b Learning Git Diff
4a4e1d5 Day 4 (DSA): Hash Set
f8d2bb4 Day 4 (C++): Input, Output
```

This is the most commonly used command for quickly viewing project history.

---

# 5️⃣ Viewing Limited Commits

View only the latest commits.

---

### Latest Commit

```bash
git log -1
```

---

### Last Three Commits

```bash
git log -3
```

---

### Last Five Commits

```bash
git log -5
```

---

### Compact Version

```bash
git log -5 --oneline
```

# Key Takeaways

- Git tracks every change made to a project.
- `git diff` helps compare file changes before committing.
- `git diff --staged` previews staged changes.
- `git log` displays the complete commit history.
- `git log --oneline` provides a concise overview of commits.
- Every commit has a unique hash, author, timestamp, and message.
