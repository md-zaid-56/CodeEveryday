# 🧠 Problem-Solving Patterns Learned

Instead of memorizing solutions, I am learning reusable patterns.

### 🔹 Hashing

Useful for fast lookup and frequency-based problems.

### 🔹 Two Pointers

```text
Left →       ← Right
```

Useful for:

- Array Reversal
- Palindrome Problems
- String Problems
- Sorted Array Problems

### 🔹 Binary Search

```text
Search Space
     ↓
Divide by Half
     ↓
Eliminate Half
     ↓
Repeat
```

Typical complexity:

```text
O(log n)
```

### 🔹 Reverse Traversal

```text
Start From End
      ↓
Skip Unwanted Data
      ↓
Process Required Data
      ↓
Stop at Boundary
```

### 🔹 Brute Force → Optimization

I am training myself to first understand a simple solution and then ask:

> Can I reduce the time complexity?

> Can I reduce the space complexity?

> Is there a reusable pattern hidden in this problem?

# 🎤 Interview Preparation

For every important concept, I am also practicing:

- 🧠 Conceptual Questions
- 🔥 Tricky Interview Questions
- 🐛 Debugging Questions
- 💾 Memory-Based Questions
- ⏱️ Time & Space Complexity

Example:

```text
Beginner Question:

"What is __init__()?"

Better Question:

"Does __init__() actually create the Python object?"

Deeper Understanding:

__new__() → Creates the object

__init__() → Initializes the object
```

This is the level of understanding I am working toward.
